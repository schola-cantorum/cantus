## 1. CLI module 與 entry point scaffolding（落實 Requirement「cantus ships a `cantus` console script and `python -m cantus` entry point」、呼應 Decision「CLI module layout uses single `cantus/cli.py`」與「`cantus/__main__.py` 直接 import 並呼叫 `cantus.cli.main`」）

- [x] 1.1 為 Requirement「cantus ships a `cantus` console script and `python -m cantus` entry point」呼應 Decision「CLI module layout uses single `cantus/cli.py`, not `cantus/_cli/` package」建立 `cantus/cli.py`：定義 `main(argv: list[str] | None = None) -> int` 函式作為 console_scripts entry，內部用 `argparse.ArgumentParser` 建主 parser 與一個 `serve` 子 parser（保留「`cantus <subcommand>`」的結構讓後續可擴充），現階段 `serve` 是唯一子命令；不拆 `cantus/_cli/` sub-package（YAGNI，一個子命令不值得 sub-package）；未知子命令交由 argparse 自然處理（exit 2）；驗證：`python -c "from cantus.cli import main; print(main(['--help']))"` 印出 `usage: cantus`-起首文字，無 import error
- [x] 1.2 [P] 為 Requirement「cantus ships a `cantus` console script and `python -m cantus` entry point」建立 `cantus/__main__.py`：內容為 `from cantus.cli import main; raise SystemExit(main())`，不重複 argparse 邏輯；驗證：`python -m cantus --help` 輸出與 `python -c "from cantus.cli import main; main(['--help'])"` byte-identical 且 exit 0
- [x] 1.3 [P] 為 Requirement「cantus ships a `cantus` console script and `python -m cantus` entry point」更新 `pyproject.toml`：新增 `[project.scripts]` 區段含一條 `cantus = "cantus.cli:main"`；不動既有 metadata / extras / build system；驗證：`grep -F 'cantus = "cantus.cli:main"' pyproject.toml` 命中、`pip install -e .` 後 `which cantus` 命中 venv 內的 entry point

## 2. argparse surface 與 args 解析（落實 Requirement「serve subcommand accepts host, port, registry-import, auth-mode, dashboard, and channels arguments」、呼應 Decision「`--auth-mode` 接受 lower-kebab string」與「`--dashboard` / `--no-dashboard` mutually exclusive boolean」與「`--channels` 暫時只接受 dotted-import 語法」）

- [x] 2.1 為 Requirement「serve subcommand accepts host, port, registry-import, auth-mode, dashboard, and channels arguments」在 `cantus/cli.py` 的 `serve` 子 parser 加完整 6 個 args：`--host`（str, default None）、`--port`（int, default None）、`--registry-import`（str, **required**）、`--auth-mode`（choices=`["none", "bearer", "api-key"]`, default None）、`--dashboard` / `--no-dashboard`（mutually exclusive group with `default=None`）、`--channels`（nargs=`"+"`, default None）；驗證：`cantus serve --help` stdout 同時含 `--host` / `--port` / `--registry-import` / `--auth-mode` / `--dashboard` / `--no-dashboard` / `--channels` 七個 literal 子字串
- [x] 2.2 為 Requirement「serve subcommand accepts host, port, registry-import, auth-mode, dashboard, and channels arguments」驗證 missing required 的行為：跑 `cantus serve` 不帶 `--registry-import`，stderr 含 `--registry-import` 字樣、exit code 2（argparse 自然行為）；驗證：unit test `tests/cli/test_serve_args.py::test_missing_registry_import_exits_2` 對著 `subprocess.run(["cantus", "serve"])` assert exit code 2 並 stderr 包含 `--registry-import`
- [x] 2.3 [P] 為 Requirement「--auth-mode maps kebab-case CLI values to AuthMode enum」在 `cantus/cli.py` 加 `_AUTH_MODE_BY_CLI_VALUE: dict[str, AuthMode]` 映射 `{"none": AuthMode.NONE, "bearer": AuthMode.BEARER, "api-key": AuthMode.API_KEY}`；CLI 解析後若 `args.auth_mode is not None` 就 lookup map 並指派給 `settings.auth_mode`；驗證：unit test `tests/cli/test_auth_mode_mapping.py` 對 `cantus serve --registry-import x:y --auth-mode bearer` 在 mock `cantus.serve` 下 assert `Settings.auth_mode == AuthMode.BEARER`、對 `--auth-mode api-key` assert `Settings.auth_mode == AuthMode.API_KEY`，並對 `--auth-mode BEARER`（大寫）assert exit code 2

## 3. Settings override 與 registry-import 解析（落實 Requirement「Settings override precedence is CLI args, then env vars, then Settings defaults」與 Requirement「--registry-import resolves a Registry instance from a dotted module path」、呼應 Decision「預設值來源優先順序：CLI args > env vars > `Settings` 預設」與「`--registry-import` 語法為 `module.dotted.path:variable`」）

- [x] 3.1 為 Requirement「Settings override precedence is CLI args, then env vars, then Settings defaults」在 `cantus/cli.py` 加 helper `_apply_override(settings: Settings, args: argparse.Namespace, settings_attr: str, cli_attr: str) -> None`：只在 `getattr(args, cli_attr) is not None` 時 `setattr(settings, settings_attr, value)`；不在 argparse default 寫死 env 值（保持可 monkeypatch）；驗證：unit test `tests/cli/test_settings_precedence.py` 跑 3 個 scenario（CLI > env / env > default / 全部 unset）對應 spec.md 的 precedence table 三條 row
- [x] 3.2 為 Requirement「--registry-import resolves a Registry instance from a dotted module path」在 `cantus/cli.py` 加 `_resolve_registry_import(spec: str) -> Registry`：用 `module, _, name = spec.partition(":")`，呼叫 `importlib.import_module(module)` 再 `getattr(mod, name)`，並 `assert isinstance(result, Registry)`；任一步失敗 raise 自製 `RegistryImportError` 含原始 exception 訊息；驗證：unit test `tests/cli/test_registry_import.py` 覆蓋 valid 路徑、缺 module、缺 attr、attr 非 Registry instance 四條 scenario
- [x] 3.3 [P] 為 Requirement「--registry-import resolves a Registry instance from a dotted module path」加 fixture `tests/cli/fixture_registry.py` 含一個 `registry = Registry(...)` top-level binding 供其它測試 import；驗證：`tests/cli/test_registry_import.py::test_valid_import_resolves_registry` 跑 `_resolve_registry_import("tests.cli.fixture_registry:registry")` 應回 `Registry` instance

## 4. Error handling 與 exit codes（落實 Requirement「CLI exit codes follow argparse error / cantus error / signal convention」、呼應 Decision「exit code 約定」）

- [x] 4.1 為 Requirement「CLI exit codes follow argparse error / cantus error / signal convention」在 `cantus/cli.py` 的 `main()` 用 try/except 包住 `_resolve_registry_import` / `validate_auth_config` 兩個會抛的呼叫；catch `RegistryImportError` → 印 `cantus serve: error: cannot import registry from '<spec>': <reason>` 到 stderr、return 1；catch `ValueError` from `validate_auth_config` → 印對應 message 到 stderr、return 1；catch `ImportError` from `import uvicorn` → 印 `cantus serve: error: cantus[serve] not installed. Run: pip install cantus-agent[serve]` 到 stderr、return 1；驗證：unit test `tests/cli/test_exit_codes.py` 覆蓋三條 cantus-internal error path 的 exit code = 1
- [x] 4.2 [P] 為 Requirement「CLI exit codes follow argparse error / cantus error / signal convention」驗證 SIGINT / 正常 shutdown 路徑：在 `cantus/cli.py` 不 catch `KeyboardInterrupt`，讓 uvicorn 自己處理 graceful shutdown；驗證：unit test `tests/cli/test_signal_handling.py::test_keyboard_interrupt_exits_0` 在 subprocess 內跑 `cantus serve --registry-import tests.cli.fixture_registry:registry --port 18081`，sleep 1s 後 SIGINT、assert exit code 0
