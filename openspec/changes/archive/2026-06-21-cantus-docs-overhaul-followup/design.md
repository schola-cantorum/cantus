## Context

`cantus-docs-overhaul` (archived `2026-06-21-cantus-docs-overhaul`) built the bilingual VitePress site at `docs/site/`, the generated NotebookLM corpus at `docs/api/`, and the interactive manual, then chose "Removed: none" so it would not break links that spec-pinned migrations, the spec-pinned desktop quickstart, and two runtime channel docstrings hold into the legacy `docs/*.md` tree. That left the legacy English pages as superseded duplicates of the site, the NotebookLM corpus hard to find next to the similarly-named `notebooks/` Colab directory, and the human sign-off steps uncollected. This change closes those three gaps without re-opening the runtime or `@trace`-heavy spec surface.

Hard constraints inherited from the current specs:
- `cantus-distribution` pins the README presentation contract (banner, dynamic PyPI badge, ECL-2.0 badge, Colab CTA, language switch) and requires both READMEs link to the site.
- `cantus-i18n-docs` requires every Zone A repository markdown document fall into exactly one of four layers, and `/spectra-audit cantus-docs-i18n-baseline` (Gate 1) reports any unclassified Zone A document.
- `api-docs` owns the generated `docs/api/` corpus; it is regenerated by `scripts/gen_docs_api.mjs` and guarded by a CI `git diff --exit-code docs/api/` check.

## Goals / Non-Goals

**Goals:**
- Eliminate the duplicated legacy English `docs/*.md` content while keeping every inbound link resolving.
- Make `docs/api/` discoverable as the NotebookLM corpus and disambiguate it from `notebooks/`.
- Ship a durable, version-controlled checklist of the documentation steps that require a human.

**Non-Goals:**
- No runtime code changes, no `docs/api/` generator changes, no moving of spec-pinned canonical files or the wiki suite.
- No version bump, PyPI release, or Cloudflare Pages deployment automation.

## Decisions

### Delete true orphans, redirect-stub the link-load-bearing pages

A legacy page is **deleted** only when its sole live inbound references are the README (which this change updates). A legacy page is **redirected to a one-line stub** when a spec-pinned migration, the spec-pinned desktop quickstart, a runtime docstring, or `CONTRIBUTING`/`notebooks` link into it, so the link keeps resolving. A legacy page is **kept untouched** when it has no site twin or is itself spec-pinned.

- Delete: `docs/overview.md`, `docs/core/agent.md`, `docs/core/event-stream.md`, `docs/core/inspector.md`.
- Redirect-stub (has a `docs/site/` twin and at least one non-README inbound link): `docs/quickstart.md`, `docs/tui.md`, `docs/protocols/{adapters,analyzer,debug,identity,memory,serve,skill,validator,workflows}.md`, `docs/cookbook/{errors,patterns,tips}.md`, `docs/cookbook-{line,telegram,discord,google-chat}-channel.md`.
- Keep untouched: `docs/protocols/adapters-batch2.md`, `docs/protocols/adapters-batch3.md` (no site twin), `docs/quickstart-desktop.md`, `docs/migrations/*` (spec-pinned), `docs/llm_wiki/*` (wiki-suite owned).

### Redirect stub format

Each stub replaces the page body with a short notice and a repository-relative Markdown link to the corresponding `docs/site/` source file (which renders on GitHub and resolves offline). The stub SHALL NOT hardcode a Cloudflare Pages production URL, because the site domain is bound by an operator after this change and is not known here. A stub keeps the page's original H1 so inbound anchor-free links still land on a titled page.

### README Documentation section rework

Both READMEs already lead their Documentation section with the site; this change removes the "raw Markdown also lives in `docs/`" per-page deep links into the deleted/stubbed pages and keeps the genuinely `docs/`-only resources (`llms.txt`, `docs/llm_wiki/index.md`). The banner, PyPI badge, ECL-2.0 badge, Colab CTA, and language-switch links are preserved verbatim; the `cantus-distribution` README scenarios are the gate.

### NotebookLM signpost in docs/llms-txt.md

`docs/api/` is regenerated and CI-diff-guarded, so no file is hand-placed there. The "how to feed NotebookLM" guidance, the `notebooks/` (Colab) versus `docs/api/` (corpus) disambiguation, and the fix for the two broken path references all land in `docs/llms-txt.md`; `notebooks/README.md` gains a one-line cross-link.

### Sign-off checklist classification

`docs/DOCS_RELEASE_SIGNOFF.md` is an operator runbook, English-only, single-language for operational reasons — the same category as `AGENTS.md`. It is added to the `cantus-i18n-docs` Excluded-from-translation enumeration so Gate 1 does not report it as unclassified and does not demand a zh-TW companion.

## Implementation Contract

**Observable behaviour:**
- Every legacy page named for redirect resolves to a file whose body is the redirect stub; the four deleted pages no longer exist.
- For every redirect stub, each pre-existing inbound link from a migration guide, `docs/quickstart-desktop.md`, a runtime docstring, `CONTRIBUTING.md`/`CONTRIBUTING.zhTW.md`, or `notebooks/README.md` still resolves to an existing file.
- `npm run docs:build` exits 0 with both locales; `npm run docs:api` leaves `git diff --exit-code docs/api/` clean (the generator is not touched).
- `docs/llms-txt.md` contains no reference to `docs/llms.txt` or `docs/api/llms-txt.md`; it names `docs/api/` as the NotebookLM upload target and states `notebooks/` is Colab execution.
- `docs/DOCS_RELEASE_SIGNOFF.md` exists with the six human steps and is enumerated in the `cantus-i18n-docs` Excluded-from-translation layer.

**Scope boundaries:**
- In scope: the legacy `docs/*.md` pages listed above, `README.md`, `README.zhTW.md`, `docs/llms-txt.md`, `notebooks/README.md`, `docs/DOCS_RELEASE_SIGNOFF.md`, and the two spec deltas.
- Out of scope: `cantus/` runtime, `scripts/gen_docs_api.mjs`, `docs/api/**`, `docs/site/**` content, `docs/migrations/*`, `docs/quickstart-desktop.md`, `docs/llm_wiki/*`, version files, deployment.

## Risks / Trade-offs

- [Deleting `docs/overview.md` dangles a historical `CHANGELOG.md` mention] → accepted: CHANGELOG entries are immutable history; the only live inbound link (README) is updated in this change.
- [README rewrite breaks the presentation contract] → run the `cantus-distribution` README banner scenarios as the gate, substring by substring.
- [Stub loses a deep anchor a migration linked to] → accepted: migration guides are historical; the stub keeps the page title and forwards to the full site page.
- [Gate 1 flags the new checklist as unclassified] → mitigated by the `cantus-i18n-docs` MODIFIED delta adding it to Excluded-from-translation.
- [MODIFIED requirement eats its trailing `@trace` at archive (recurring)] → the `cantus-i18n-docs` delta reproduces the existing `@trace` block verbatim; grep the `@trace` count before and after archive and restore any eaten or orphaned block.
